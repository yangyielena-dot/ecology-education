## 二、三大智能体模块

### 1. 重建生态星球 (planet)

**角色**: 专业的生态教育AI助手，引导学生创造自己的生态星球

**核心流程**:
```
阶段一：概念教学
├── 概念1：种群 → "生活在同一地方的同一种生物"
└── 概念2：生态系统 → "生物+周围环境"构成的整体

阶段二：创意设计
├── 策略1：生态关系引导 → 引导思考生物间关系
├── 策略2：环境完善引导 → 引导思考环境需求
├── 策略3：生态链扩展引导 → 引导添加更多生物
├── 策略4：星球完整性引导 → 引导完善生态系统
└── 策略5：应对"不知道" → 给思考方向而非答案
```

**关键技术点**:
- 检查对话历史判断当前阶段，避免重复讲解
- 支持"重新开始"指令，清除历史重新教学
- 核心原则：关注生态关系，不盯单个生物

---

### 2. 生态瓶设计 (bottle)

**角色**: 生态瓶设计AI助手"苗苗"，帮助学生设计生态瓶

**核心功能**:
```
水生生态瓶：
├── 溶氧量 → 光照影响水草产氧
├── 废物 → 动物产生废物，植物和螺帮助清理
└── 稳定性 → 整体生态平衡程度

陆生生态瓶：
├── 湿度 → 苔藓增加湿度，光照过长会降低
├── 有机物 → 蚯蚓和鼠妇增加有机物
└── 稳定性 → 整体生态平衡程度
```

**对话策略**:
- 引导学生观察数据指标，自己发现问题
- 用问句结尾，不直接给答案
- 永远引导学生思考，不告诉"你的问题是..."

---

### 3. 生态瓶小侦探 (detective)

**角色**: AI侦探助手"波波"，帮助学生诊断生态瓶疾病

**三大病例**:

| 病例 | 病因 | 症状 | 治疗方案 |
|------|------|------|----------|
| 浮头病 | 水草太多 + 光照时间短 | 下午溶氧量低(2.8mg/L) | 减少水草，增加光照 |
| 绿水病 | 光照太强 + 投喂太多 | 废物浓度高(25mg/L) | 减少光照，少喂食物 |
| 死寂病 | 鱼太多 + 水草太少 | 溶氧量极低(0.8-2.0mg/L) | 减少鱼，多种水草 |

**对话流程**:
```
第一步：观察症状 → "小鱼怎么了？水是什么颜色？"
第二步：找原因 → 用选择题或提示引导
第三步：给提示 → 用生活化比喻解释
第四步：引导治疗 → 肯定学生，引导点击治疗按钮
```

## 三、技术实现

### 3.1 API 接口 (`/api/chat/route.ts`)

```typescript
// 1. 接收请求参数
const { message, context = 'planet', conversationHistory = [] } = await request.json();

// 2. 构建对话历史
const messages = [
  { role: 'system', content: SYSTEM_PROMPTS[context] },
  ...conversationHistory.slice(-10),  // 保留最近10轮
  { role: 'user', content: message }
];

// 3. 调用大模型（流式响应）
const stream = client.stream(messages, {
  model: 'doubao-seed-1-8-251228',
  temperature: 0.7,
});
```

### 3.2 前端对话组件

```typescript
// 位置: src/app/detective/case/[id]/page.tsx
```

### 3.3 学习记录 (Supabase)

```sql
-- learning_sessions: 记录学习会话
- id: 会话ID
- student_id: 学生ID
- module_type: 模块类型 (planet/bottle/detective)
- module_detail: 模块详情 (病例ID等)
- status: 会话状态
- score: 得分
- started_at / ended_at: 时间

-- learning_messages: 记录对话历史
- id: 消息ID
- session_id: 关联会话
- role: user/assistant
- content: 消息内容
- metadata: 附加数据
```

